概述：远程OTA采用网络调试助手向网关发送升级链接，网关接收到升级链接后下载对应固件到开发板中。
准备工具：1.dtu网关（最好有网关配置界面） 2.stm32F4开发板 3.网络调试助手 

升级方式：
1.烧录BL、APP1、APP2到开发板中
		现象：串口输出如图信息
			![[远程OTA升级方式-1777017826804.webp|404x322]]

> [!NOTE]
> 1.3000ms内发送字符U（不区分大小写）进入串口升级而非远程ota
> 2.“本次升级目标：APP2 ···”，即固件升级区域为APP2位置，固件也要指向此区域，否则会升级失败

2.准备新固件
	获取bin文件方式（bin文件保存在相应目录中）：
	![[远程OTA升级方式-1777019492456.webp|557x417]]
	后续bin文件获取只需`编译`即可，不需要再下载到板子上了，用bin文件进行远程ota升级，而非通过`编译器下载`
	在固件和python脚本(计算 bin 文件的 CRC32，并直接将这 4 个字节追加到 app.bin 的最末尾)放在同个文件夹执行：`py append_crc32.py App1.bin App1_ota.bin`
	在`App1_ota.bin`存放的文件夹中执行：`py -m http.server 8000` 
	在cpolar web ui中创建本地串口映射至公网的端口：
	![[远程OTA升级方式-1777020638473.webp]]

> [!NOTE]
> 8000是bin文件的本地端口，开放到11422的公网地址。
> 8080是网络调试助手的本地端口，网关连接12998的公网地址。
> 网络调试助手发送 `http://17.tcp.cpolar.top:11422/App1_ota.bin`（即ota文件的公网地址）到网关。

3.配置DTU网关
	网关型号：TAS-LTE-892D
	配置内容：
	![[远程OTA升级方式-1777019248150.webp|381x386]]
	
> [!NOTE] 
> 配置完点击：进入配置状态→一件配置参数→退出配置状态→重启设备
>如果出现网关不正常：调整串口波特率重新连

4.配置网络调试助手
	配置方式：
	![[远程OTA升级方式-1777020807767.webp]]
	网关重启后，点击`发送`，网关会接收到该URL链接并重启，重启之后下载bin文件，开发板会对bin文件进行CRC校验，下载并校验成功则升级成功，再次`复位`后开发板进入并启动新固件。升级完成！
